## 5 種 Alert 類型

| 類型 | 條件 | severity | 觸發時機 |
|---|---|---|---|
| `bias_changed` | bias 升降級且涉及 BUY/HOLD | up→BUY=high,其他=medium | 持股決策變動 |
| `disqualifier_triggered` | BUY/HOLD → OUT_OF_CIRCLE | high | 結構性風險 |
| `new_top10` | 今天首次擠進 BUY 前 10 | medium | 新候選人 |
| `dropped_from_top10` | 昨天前 10、今天跌出 | low | 情報追蹤 |
| `mos_first_positive` | DCF 安全邊際從非正轉正 | high | 便宜進場時機 |

**訊噪比設計**:
- WATCH ↔ AVOID 邊界外移動不 alert(訊號弱)
- 新 ticker 第一次出現不算變動(沒昨天可比)
- `format_alert_message()` 只在 high/medium 推送 lobby(low 全部抑制)

## P0-2 個股 thesis tracking — ✅

### 設計

| 元件 | 職責 |
|---|---|
| `agent/thesis.py` | `Thesis` / `Condition` / `ThesisStatus`、`process()` 主流程、JSON IO |
| `agent/llm.py::write_thesis()` | OpenRouter 寫 200 字 thesis,失敗 fallback template |
| `agent/diff.py::thesis_broken_alerts()` | 6th alert 類型,severity=high |
| `output/theses/<TICKER>.json` | 每檔一個檔,committed 留審計痕跡 |

### 機制

- **第一次 BUY** → LLM 寫 thesis (失敗 fallback 模板)
- **後續每天** → 機械式檢驗(不打 LLM 省成本)
- **任一 condition 違反** → `thesis_broken` alert (high severity)
- **缺值寬容**:抓不到資料的條件視為通過(避免 yfinance 抖動誤觸發)
- **動態條件合成**:roe ≥ 0.8 鎖底線、berkshire_holds 鎖狀態、industry_class 鎖分類

### 4 種 thesis state

- `new`:今日首次寫
- `valid`:條件持續成立
- `broken`:條件被違反 → 觸發 alert
- `skipped`:不在 BUY 候選(無 thesis、不算)

## P0-3 估值 ensemble — ✅

### 設計

| Method | 公式 | 哲學 |
|---|---|---|
| `dcf_two_stage` | 10y owner earnings 折現 + Gordon terminal | Buffett 自己 + 經典金融 |
| `shiller_pe` | 5y avg EPS × fair PE 17 | Graham/Shiller 風格,smooth 出週期波動 |
| `owner_earnings_yield` | 5y OE / market_cap vs treasury+6% | Buffett 公開講過的「比零息債」方法 |

Ensemble 取 3 個 intrinsic 的 min/median/max,MOS 用 mid。

### 5 桶 Consensus 分類

| Mid MOS | Consensus |
|---|---|
| ≥ +30% | very_cheap |
| ≥ +10% | cheap |
| (-20%, +10%) | fair |
| (-50%, -20%] | expensive |
| ≤ -50% | very_expensive |

### 實測

| Ticker | DCF | Shiller PE | OE Yield | Consensus | Mid MOS |
|---|---:|---:|---:|---|---:|
| AAPL | $95 | $109 | $67 | very_expensive | -185% |
| KO | $26 | $42 | $18 | very_expensive | -207% |
| MCO | $234 | $180 | $107 | very_expensive | -156% |
| **BAC** | $54 | $63 | $38 | **fair** | **+3%** |
| **JPM** | $289 | $301 | $179 | **fair** | -7% |
| NVDA | — | $87 | — | very_expensive | -132% |

3 個模型互相印證;銀行業 BAC/JPM 同時 fair = 強訊號,Buffett 重倉股 AAPL/KO/MCO 仍 very_expensive 是市場現實。


LLM 自動引用「估值模型分歧大」+ ROE 持續性,證明 ensemble context 確實被使用。

## P1-2 護城河結構化 — ✅

### 設計

5 個 Buffett + Pat Dorsey 護城河類型,每種有量化代理指標 (0-10):

| 類型 | 量化代理 |
|---|---|
| intangible_assets | 高毛利 + 高淨利 + ROE 持續性 |
| switching_costs | ROE 持續性 + 5y ROIC + 5y OE margin |
| network_effects | 規模 + 高成長 + 高毛利 + 高 ROIC |
| cost_advantage | 規模 + 高毛利 + 高買回率(現金過剩) |
| efficient_scale | 監管行業 + 穩定 ROE + 規模 |

### 整合輸出

- `MoatProfile`:5 components + overall_score + strength (strong/moderate/weak) + dominant_types + trend
- 趨勢分析:近 5y ROE avg vs 前 5y ROE avg → widening / stable / narrowing

### 改動

| 檔案 | 動作 |
|---|---|
| `agent/moat.py` | 新增,5 score_*() + analyze_trend + evaluate() |
| `agent/verdict.py` | Verdict 加 `moat`、rationale 加「🏰 護城河結構化評分」 |
| `agent/llm.py` | system prompt 加 moat 引用規則、user prompt 加 moat 區塊 |
| `src/build_scan_html.py` | JSON 加 `moat` |
| `agent/tests/test_moat.py` | 17 個測試 |

### 實測

| Ticker | Strength | Score | Trend | Dominant |
|---|---|---:|---|---|
| MCO | strong | **9.0** | stable | intangible + switching |
| V | strong | **9.0** | widening | intangible + switching |
| GOOGL | strong | 8.6 | widening | intangible + network |
| AAPL | strong | 7.6 | widening | network + switching |
| NVDA | strong | 7.2 | widening | network + intangible |
| KO | strong | 7.0 | widening | intangible + switching |
