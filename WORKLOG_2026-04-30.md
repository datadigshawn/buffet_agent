## Phase 4-B4 — 行業特化規則 + 金融業 XBRL 概念修復

### (1) 金融業 ROE/Equity 概念順序修正

**問題**:V 在 2009-2011 用 `StockholdersEquity`,2012+ 改用 `StockholdersEquityIncludingPortionAttributableToNoncontrollingInterest`。`get_concept_units()` 卡在第一個 match → 只拿 3 年資料 → ROE 持續性誤判 0.00。

**修正**:`agent/sources/sec.py` `CONCEPT_ALTERNATES["TotalEquity"]` 把「Including NCI」放第一順位,V 拿到 17 年資料 → ROE 持續性 1.00。

| `src/build_scan_html.py` | JSON 輸出 `intrinsic_value_per_share`、`margin_of_safety_pct`、`qualitative`、`industry_class` |

### DCF 設計重點

- Stage 1 (年 1-10): 用近 5 年 owner earnings CAGR,**封頂 15%**(避免過度樂觀),**下限 2.5%**
- Stage 2 (terminal): 永續成長 2.5% (~長期 GDP)
- 折現率預設 10% = 10y treasury 4% + Buffett 風險溢酬 6%(env `BUFFET_DCF_DISCOUNT_RATE`)
- 銀行/保險:OE = NetIncome (這類業務無實體 capex,NI 近似可分配現金)
- Capex 缺資料 → fallback 到 NetIncome

### LLM stub 設計

- `BUFFET_LLM_BACKEND` env (預設 `none`) → 將來切 hermes/minimax 不用改 verdict.py
- `judge(ticker, context)` 介面已定型,context 包含:量化分數、SEC 持續性、DCF MOS、KB 概念
- 對 BUY/HOLD 才呼叫(省成本),其他 bias 跳過

### 設計

- **純 stdlib urllib**,無新依賴;模式參考 war-room 但獨立實作
- **JSON-only prompt**:強制回 7 欄位 JSON,容錯 parse 處理 ```json fence
- **預設模型** `minimax/minimax-m2.5`(沿用 war-room 慣例),可 `BUFFET_LLM_MODEL` env 切
- **`reasoning.exclude=True`** 讓 MiniMax 直接吐 content,不耗 token 在 reasoning trace
- **C4 併入 C3**:同一次 API 呼叫就拿到 management_grade + moat + 能力圈 + recommendation,省成本
